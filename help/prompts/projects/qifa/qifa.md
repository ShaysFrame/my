# Project: [Qifa]

## 1. What was the problem it solved / who was the customer?

Qifa is a project that's probably the biggest project for me because I made it from scratch from flutter's myapp to the where we are now in the developement phase, as a single developer. there are 5 people, one is the with the idea, and I'm the one who's building it, the idea was from Gavin who has 33% or maybe even more idk how much for sure, of the company share, that we should make a platform for the salons. Where people will be able to use app to search for shops nearby, like didi, and so on, and then have a good service, and the company will have some sort of insurance service like if the user buys them there will be some sort of guarentee or something.The insurance will give them some sort of guarentee that'd give them compensation if the service was not satisfactory. The app is still not complete, and I still need to configure the we chat's pay system. 

There are many features in the app I kinda forgot, but it solves the core above problem where it's a app for salons, where all the salons if they wanted can register with the app, and start receiving orders, user will be able to look/search for near by shops, with in their specified search filters such as 5km radius or something, then the shop's app will get some notification. There are main two apps, one is Qifaclient, another is qifashop, the qifa shop will queue the users, and the shop will receive the queue, and then shop will have some time window to response in order to successfully, accept a user's request to a service, in the acceptance response they'll be able to add services provided for the request, and then further details if needed, and the most importantly the amount they'll be charged for the service. And also they'll include, whether the "shop insurance" is enabled for this service or not, note that there are "qifa insurance" which is platform and also "shop insurance" from the shop's side which shop can control. Oh I almost forgot that they can also assign workers they have available for the service, the best and the suitable one, but if they don't deside one then the app will let the user select I think, I kinda forgot which party selects the worker, I think it's the client side, I'll check on that later. 

Once the shop accepts the response, and sends their feedback for the service request, the user will also get a feedback in their app, where the user would then be able to see all the near by shop who had responded to their service request, with details such as service name, and the full details for the services that it includes, and also the prices, and insurance info and so on. User will have a screen where they'll be able to see a list of shop that has responded with their offer, the the client will be able to select the offer and then further customize as needed, time, select worker, use coupons, and so on.
